# Alignment Memo Input — AI Adoption Framework

Hey, I want to put together a framework for how we think about AI tools across product and engineering. Here's where my head is at:

---

## What's Happening

We should start using AI tools across the team. Everyone's talking about it, other companies are doing it, and I think we're falling behind if we don't get on this.

Some teams are already using Copilot — the payments backend team has been using it for a few months and they say it saves them time on boilerplate. But other teams haven't touched it. The design team tried Midjourney for some explorations and the VP of Design shut it down because of IP concerns. A few PMs are using ChatGPT to draft documents but nobody talks about it openly because they're not sure if it's allowed.

Our CTO mentioned in the last all-hands that he's "excited about AI" and wants us to "be thoughtful about how we adopt it." That was two months ago and nothing has happened since.

---

## What I Think We Should Do

I think we need a framework that helps teams figure out when and how to use AI tools. We should be thoughtful about it. AI should help, not replace. We want to use it responsibly.

Here are some tools I've been looking at:
- GitHub Copilot for code generation
- Cursor for AI-assisted development
- ChatGPT / Claude for document drafting, research, summarization
- v0 and Bolt for rapid prototyping
- Figma AI features for design

I think we should evaluate each tool, maybe create a scoring rubric, and then roll it out across the org.

---

## Some Concerns

- Legal hasn't weighed in yet on data privacy — we handle financial data and some of these tools send data to external APIs
- Some engineers are worried about code quality and "AI slop"
- The VP of Design's reaction to Midjourney makes me think there's resistance in some parts of the org
- We don't have budget allocated for AI tools this quarter

---

## What I Need

Can you draft this into a proper framework document? I want something we can circulate that helps everyone understand how to think about this. We should be thoughtful and responsible about it.
